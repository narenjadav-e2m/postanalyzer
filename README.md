# PostAnalyzer

> Automated SEO, accessibility, and content QA audits for WordPress posts — with AI-powered, one-click field fixes.

## What it does

PostAnalyzer adds a dedicated admin screen (**Post Analyzer**, also reachable from the admin bar) where an editor can pick any post or page and run a full audit. For the selected post it reports:

- **Basic info** — title, slug, excerpt, author, published / updated dates, status, post type, categories, tags, and a Unicode-aware word count (counts CJK and other non-Latin scripts correctly).
- **SEO data** — pulled from **Rank Math** meta (`rank_math_title`, `rank_math_description`, `rank_math_focus_keyword`, `rank_math_robots`): SEO title, meta description, focus keyword(s), index/follow flags, a list of detected issues (missing/too-short/too-long title & description), and a computed **SEO score (0–100)**.
- **Featured image & attached images** — discovered from the featured thumbnail, parent-attached media, inline `<img>` tags, and Gutenberg blocks (`core/image`, `core/gallery`, `core/media-text`), with de-duplication. Each image shows alt text, title, caption, dimensions, file size, MIME type, and a **missing-alt-text badge**.
- **AI suggestions** — 5–10 specific, actionable improvement suggestions plus SEO-friendly URL slug suggestions, generated by the configured AI platform.

### Inline AI-assisted editing

From the report, editable fields show an **edit popover** that can:

1. **Suggest** — ask the AI for replacement options tailored to the post (and image, for alt/title). Length-sensitive SEO fields are reordered so options within Rank Math's recommended character range appear first.
2. **Update** — write the chosen value straight back to WordPress.

Editable fields (strict allow-list — the routes can never write arbitrary meta): `post_title`, `post_excerpt`, `slug`, `seo_title`, `seo_description`, `focus_keyword`, image `alt`, and `image_title`. Writes are gated by the per-post / per-attachment `edit_post` capability.

## Requirements

| Requirement | Minimum |
|---|---|
| WordPress | 6.3+ |
| PHP | 8.1+ |
| Rank Math SEO | Required (SEO fields read/write `rank_math_*` meta) |
| Node.js | 18+ (for development) |

## Installation

1. Upload the `postanalyzer` folder to `/wp-content/plugins/`
2. Activate the plugin via **Plugins → Installed Plugins**
3. Go to **Post Analyzer → Settings** and configure your AI platform API key
4. Select a post and click **Analyze Post**

## AI Platforms Supported

Configure one platform and its API key in Settings. Keys are validated live against the provider and stored **encrypted** (AES-256-CBC with a random IV, keyed from `wp_salt`); only *which* platforms have a key is ever exposed back to the UI.

| Platform | Model | Notes |
|---|---|---|
| **Groq** | llama-3.3-70b-versatile | Recommended — fast & free tier |
| **OpenAI** | gpt-4o-mini | Reliable, paid |
| **Gemini** | gemini-2.0-flash | Google AI Studio free tier |

## Development Setup

```bash
cd postanalyzer
npm install
npm run build        # One-time production build
npm run watch        # Watch mode during development
```

The PHP autoloader is generated via Composer (`composer install`). Build assets are emitted to `build/` and enqueued with mtime-based cache-busting.

## Extending

### Add a custom post type to the analyzer

```php
add_filter( 'postanalyzer_post_types', function( $types ) {
    $types[] = 'my_custom_post_type';
    return $types;
} );
```

Defaults to `[ 'post', 'page' ]`.

### Hook into analysis results

Future: `postanalyzer_after_analyze` action will fire with `$post_id` and `$response`.

## REST API

All endpoints are under `/wp-json/postanalyzer/v1/`.

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `POST` | `/analyze-post` | Editor+ (`edit_post` on target) | Run full analysis on a post |
| `POST` | `/suggest-field` | `edit_post` on target | AI suggestions for a single editable field |
| `POST` | `/update-field` | `edit_post` on target | Save a new value to an allow-listed field |
| `GET` | `/posts` | Editor+ | List analyzable posts |
| `GET` | `/users` | Editor+ | List eligible authors |
| `GET` | `/get-settings` | Admin | Retrieve current settings (no key material) |
| `POST` | `/save-settings` | Admin | Save AI platform & API key |
| `POST` | `/validate-key` | Admin | Validate an API key live |
